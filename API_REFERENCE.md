# 📚 API Reference - SEDERG PAY\n\n## Base URL\n```\nhttp://localhost:8000/api\n```\n\n## Autenticación\n\nTodos los endpoints protegidos requieren el header:\n```\nAuthorization: Bearer <token>\n```\n\n---\n\n## Endpoints\n\n### 🔐 Autenticación\n\n#### POST `/auth/login`\nLogin de usuario\n\n**Request:**\n```json\n{\n  \"email\": \"user@example.com\",\n  \"password\": \"password123\"\n}\n```\n\n**Response:**\n```json\n{\n  \"token\": \"eyJhbGc...\",\n  \"user\": {\n    \"id\": 1,\n    \"nombre\": \"Juan\",\n    \"email\": \"user@example.com\",\n    \"estado_kyc\": \"aprobado\"\n  }\n}\n```\n\n#### POST `/auth/registro`\nRegistro de nuevo usuario\n\n**Request:**\n```json\n{\n  \"nombre\": \"Juan García\",\n  \"dni\": \"12345678\",\n  \"email\": \"juan@example.com\",\n  \"telefono\": \"+53123456789\",\n  \"pais\": \"Cuba\",\n  \"password\": \"SecurePass123!\"\n}\n```\n\n**Response:** Igual a `/auth/login`\n\n#### GET `/auth/perfil`\nObtener perfil del usuario autenticado (Requiere token)\n\n**Response:**\n```json\n{\n  \"user\": {\n    \"id\": 1,\n    \"nombre\": \"Juan\",\n    \"email\": \"user@example.com\",\n    \"estado_kyc\": \"aprobado\"\n  }\n}\n```\n\n---\n\n### 💳 Remesas\n\n#### GET `/remesa`\nListar remesas del usuario (Requiere token)\n\n**Query Parameters:**\n- `page` (int, default: 1)\n- `estado` (string: pending, depositada, en_proceso, cobrada, rechazada)\n- `ordenar` (string: fecha, monto)\n\n**Response:**\n```json\n{\n  \"data\": [\n    {\n      \"id\": 1,\n      \"usuario_id\": 1,\n      \"destino_id\": 1,\n      \"destinatario_dni\": \"87654321\",\n      \"monto_rub\": 5000,\n      \"monto_cup\": 120000,\n      \"tasa_cambio\": 24,\n      \"estado\": \"cobrada\",\n      \"codigo_cobro\": \"RES-2026-001\",\n      \"created_at\": \"2026-05-26T10:00:00Z\"\n    }\n  ],\n  \"pagination\": {\n    \"total\": 10,\n    \"page\": 1,\n    \"limit\": 10,\n    \"pages\": 1\n  }\n}\n```\n\n#### POST `/remesa/crear`\nCrear nueva remesa (Requiere token)\n\n**Request:**\n```json\n{\n  \"destino_id\": 1,\n  \"destinatario_dni\": \"87654321\",\n  \"monto_rub\": 5000\n}\n```\n\n**Response:**\n```json\n{\n  \"remesa\": {\n    \"id\": 1,\n    \"usuario_id\": 1,\n    \"estado\": \"pendiente\",\n    \"monto_rub\": 5000,\n    \"monto_cup\": 120000\n  }\n}\n```\n\n#### GET `/remesa/{id}`\nObtener detalles de una remesa (Requiere token)\n\n**Response:** Objeto remesa completo\n\n---\n\n### 🏦 Bancos\n\n#### GET `/banco`\nListar bancos disponibles\n\n**Response:**\n```json\n{\n  \"data\": [\n    {\n      \"id\": 1,\n      \"nombre\": \"Banco Nacional de Cuba\",\n      \"codigo\": \"BNC\",\n      \"pais\": \"Cuba\"\n    },\n    {\n      \"id\": 2,\n      \"nombre\": \"Sberbank\",\n      \"codigo\": \"SBR\",\n      \"pais\": \"Rusia\"\n    }\n  ]\n}\n```\n\n---\n\n## Códigos de Error\n\n| Código | Significado |\n|--------|----------|\n| 200 | OK |\n| 201 | Creado |\n| 400 | Solicitud inválida |\n| 401 | No autenticado |\n| 403 | No autorizado |\n| 404 | No encontrado |\n| 422 | Error de validación |\n| 500 | Error del servidor |\n\n### Error Response\n```json\n{\n  \"success\": false,\n  \"message\": \"Email ya registrado\",\n  \"errors\": {\n    \"email\": [\"Email ya registrado\"]\n  }\n}\n```\n\n---\n\n## Rate Limiting\n\n- Máximo 100 requests por minuto\n- Headers retornados: `X-RateLimit-Limit`, `X-RateLimit-Remaining`\n\n---\n\n## Versión\n\n**API Version:** 1.0.0  \n**Último update:** 2026-05-26\n
