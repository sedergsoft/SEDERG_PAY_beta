# 🤝 Guía de Contribución\n\n¡Gracias por tu interés en contribuir a SEDERG PAY!\n\n## Cómo Contribuir\n\n### 1. Fork del Proyecto\n```bash\ngit clone https://github.com/tu-usuario/SEDERG_PAY_beta.git\ncd SEDERG_PAY_beta\n```\n\n### 2. Crear una Rama\n```bash\ngit checkout -b feature/tu-feature\n```\n\n### 3. Hacer Cambios\n- Sigue el estilo de código existente\n- Asegúrate de que TypeScript compila sin errores\n- Prueba tu código localmente\n\n### 4. Commit\n```bash\ngit commit -m \"feat: Descripción de tu feature\"\n```\n\n### 5. Push\n```bash\ngit push origin feature/tu-feature\n```\n\n### 6. Pull Request\n- Abre un PR con descripción clara\n- Espera revisión de los mantenedores\n\n---\n\n## Normas de Código\n\n### TypeScript\n- Usar strict mode\n- Tipos explícitos en funciones\n- Evitar `any`\n\n### Componentes React\n- Componentes funcionales\n- Custom hooks cuando sea necesario\n- Nombres descriptivos\n\n### Estilos\n- TailwindCSS utilities\n- No agregar CSS custom innecesario\n- Mobile-first approach\n\n---\n\n## Tipos de Commit\n\n- `feat`: Nueva funcionalidad\n- `fix`: Corrección de bug\n- `docs`: Documentación\n- `style`: Formato/estilos\n- `refactor`: Refactorización\n- `perf`: Performance\n- `test`: Tests\n- `chore`: Tareas\n\n---\n\n## Preguntas?\n\n- Abre una [Discussion](https://github.com/sedergsoft/SEDERG_PAY_beta/discussions)\n- Abre un [Issue](https://github.com/sedergsoft/SEDERG_PAY_beta/issues)\n\n---\n\n¡Gracias! 🙌\n
